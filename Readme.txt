*Быстрый старт (если нужно сразу запустить)*
Убедитесь, что установлены Python 3.8+, MATLAB (с Engine для Python) и ANSYS (любая версия от 18.2, с лицензией).

Положите в рабочую папку файл с конфигурациями for_ansys_1.npy (формат описан ниже) и нужные профили naca*.csv, а также
файлы из папки MATLAB.

Формат файлов конфигураций (for_ansys_X.npy)
Это NumPy-массив, где каждая строка – одна конфигурация крыла:

Индекс	Содержание
0	Площадь крыла S, м²
1	Удлинение λ
2	Сужение η
3	Стреловидность χ, град
4	Относительная толщина профиля с, (доля)
5	Крейсерская высота, Н м
6	Скорость полёта V, м/с
7	Удельная нагрузка на крыло, p_кр кг/м^2
8	Эксплуатационная перегрузка, n_y
9	Допустимое напряжение [σ], МПа
10	Положение переднего лонжерона (доля хорды)
11	Положение заднего лонжерона (доля хорды)
12	Целевой коэффициент подъёмной силы Cy
13	Скоростной напор, q Па
14	Плотность воздуха ρ, кг/м^3
15	Количество нервюр на полукрыле

Запустите главный скрипт:

python parametros.py
Он по очереди вызовет все стадии. В конце в папке "optimization_results_manual_spars/" появятся результаты для каждой конфигурации.

*Если хотите разобраться пошагово — читайте дальше.*

- Структура и логика работы
Весь процесс разбит на независимые модули, которые обмениваются текстовыми файлами.
Это сделано специально: так проще отлаживать и при необходимости заменять отдельные части (например, аэродинамику или генератор сетки).

Главный управляющий файл — parametros.py. Он просто читает for_ansys_1.npy и запускает цепочку скриптов.
Однако в реальной работе можно запускать скрипты по отдельности, чтобы контролировать каждый этап.

Ниже — описание каждого модуля так, как они используются.

1. run_aero_batch.py + папка MATLAB – аэродинамический расчёт (MATLAB)

/Для чего нужен.

Этот скрипт для каждой конфигурации из for_ansys_X.npy (X любое целое число) запускает MATLAB-функцию aero_solver,
которая считает распределение давления методом дискретных вихрей. 
В результате мы получаем файлы с силами в узлах панельной сетки — они потом пойдут на интерполяцию.

/Что подаётся на вход.

for_ansys_{file_number}.npy – массив, где каждая строка содержит параметры крыла: площадь, удлинение, сужение, стреловидность,
толщину профиля (в долях), скорость, плотность, целевой Cy и т.д. Всего не менее 12 столбцов (подробнее – в коде).

Файлы профилей naca24XX-il.csv – они подбираются автоматически по толщине из конфигурации.

/Что получается на выходе.

Для каждой конфигурации idx создаются:

{idx}_{alpha}.txt – три столбца: X, Y, сила (Н) в узлах аэродинамической сетки.
Имя файла содержит угол атаки alpha, который был подобран для достижения целевого Cy.

config_{idx}_alpha_CL.txt – две строки: alpha и Cy (для справки).

Общий файл aerodata_target.npy – сводка по всем конфигурациям: углы атаки, коэффициенты подъёмной силы и сопротивления, подъёмная сила.

/Как запустить.

python run_aero_batch.py 1
где 1 – номер набора конфигураций. По умолчанию номер = 1.

2. geom_ansys.py – генерация конечно-элементной сетки

/Для чего нужен.

Строит параметрическую КЭ-модель крыла в формате CDB (ANSYS).
В этой работе реализована трёхслойная обшивка (несущие слои + заполнитель) и кессонное закрепление на корне.

/Что подаётся на вход.

Те же for_ansys_{file_number}.npy – из каждой конфигурации берутся S, λ, η, χ, толщина профиля,
позиции лонжеронов (столбцы 10 и 11) и количество нервюр (столбец 15).

Дополнительно могут быть файлы spar_positions.txt и rib_count.txt – если они есть, их значения имеют приоритет над конфигурацией.
Это удобно, когда вы вручную хотите задать схему.

Файл профиля (например, naca2412-il.csv).

/Что получается на выходе.

wing_mesh_config_{idx}.cdb – готовая КЭ-модель для ANSYS.

{idx}-{idx}.txt – координаты центров элементов верхней обшивки. Этот файл нужен для интерполяции давлений.

Две картинки: wing_silhouette_config_{idx}.png (вид крыла в плане) и spars_ribs_top_view_config_{idx}.png (схема лонжеронов и нервюр).

/Как запустить.

Обработать все конфигурации:
python geom_ansys.py 1

Только одну (например, пятую):
python geom_ansys.py 1 5

3. Interpolador.py – перенос давлений с панельной сетки на КЭ-модель

/Для чего нужен.

Аэродинамическая сетка и конечно-элементная сетка не совпадают.
Надо переложить нагрузки так, чтобы суммарная сила сохранилась с погрешностью не более 3%.
В алгоритме используется нормализация координат (локально по хорде и глобально по размаху) и линейная интерполяция на триангуляции Делоне.

/Что подаётся на вход.

Файлы сил от run_aero_batch.py: {idx}_{case}.txt. ( {case} = alpha )

Файлы центров элементов от geom_ansys.py: {idx}-{idx}.txt.

/Что получается на выходе.

Для каждой пары создаются:

resultados_interpolacion_{idx}_{case}.txt – X, Y, Z, давление (Па) в центрах верхних элементов обшивки.

info_interpolacion_{idx}_{case}.txt – служебная информация: исходная сумма сил, проинтегрированная сила, относительная ошибка.

Общий Excel-отчёт resultados_verificacion.xlsx – сводка по всем интерполяциям.

Картинка с наложением сеток.

/Как запустить.

Все итерации:

python Interpolador.py 1

Только для определённой итерации (например, 5):

python Interpolador.py 1 5

4. nagryzka_ansys.py – формирование APDL-файла с нагрузками

/Для чего нужен.

Собирает все расчётные случаи (их может быть несколько – разные углы атаки, перегрузки) в один APDL-скрипт,
где каждый случай записывается как отдельный load step. Это позволяет ANSYS решить все задачи за один запуск, а не перечитывать геометрию каждый раз.

/Что подаётся на вход.

wing_mesh_config_{idx}.cdb.

resultados_interpolacion_{idx}_{case}.txt – для каждого случая.

/Что получается на выходе.

simple_load_{idx}.apdl – главный APDL-файл.

cases_order_{idx}.txt – порядок случаев (просто список).

(Позже, после расчёта ANSYS, появляются файлы reactions_..., G_output_..., max_uz_... – их генерирует сам APDL).
Для расчёта достаточно открыть файл simple_load_{idx}.apdl в APDL

/Как запустить.

Обработать все случаи для итерации 5:
python nagryzka_ansys.py 5

Только один случай (например, alpha = 2.5):
python nagryzka_ansys.py 5_2.5

5. thickness_optimizer.py – оптимизация толщин трёхслойных панелей

/Для чего нужен.

Это ядро всей работы. Модуль запускает ANSYS в пакетном режиме и внутри APDL выполняет итерационную оптимизацию:

Сначала увеличивает толщину для каждого элемента, пока эквивалентные напряжения по Мизесу не станут ниже допустимых.

Затем дооптимизирует только перегруженные элементы (это ускоряет сходимость).

Наконец, запускает баклинг-анализ и адаптивно увеличивает толщину заполнителя t_core в зонах, где энергия деформации моды наибольшая.

/Что подаётся на вход.

wing_mesh_config_{config_index}.cdb.

simple_load_{config_index}.apdl.

Параметры из for_ansys_X.npy (допустимое напряжение, площадь).

Множество параметров оптимизации (доли слоёв, пороги, максимальные толщины) – они задаются в конструкторе класса или через аргументы командной строки.

/Что получается на выходе.

Все результаты сохраняются в папку "optimization_results_manual_spars/config_{idx}_best/". Среди ключевых файлов:

mass_config_{idx}.txt – масса полукрыла после оптимизации (кг).

masses_by_type_{idx}.txt – массы отдельно для обшивки, лонжеронов, нервюр, задней стенки.

thickness_skin_{idx}.txt, thickness_spar_{idx}.txt, ... – для каждого элемента записаны total, t_face, t_core (в метрах).

G_before_buckling... и G_after_buckling... – силовой фактор G = ∫σ_eqv dV (Н·м) до и после баклинга.

Ck_before... и Ck_after... – безразмерный коэффициент Ck = G/(P·√S), где P – максимальная аэродинамическая сила.

buckling_factors_iter_*.txt – коэффициенты запаса по устойчивости для пяти мод на каждой итерации.

strain_energy_core_layer_config_{idx}.txt – суммарная энергия деформации заполнителя (для оценки эффективности).

/Как запустить.

Обычно скрипт вызывается из spar_optimizer.py, но можно и напрямую:

python thickness_optimizer.py 1
Можно передать до 27 параметров (например, face_ratio, core_ratio, max_buck_iter),
но я рекомендую оставить значения по умолчанию – они хорошо проверены для оптимизации конструкций).

6. spar_optimizer.py – байесовская оптимизация положений лонжеронов и числа нервюр

/Для чего нужен.

Верхний уровень. Он ищет такие положения переднего и заднего лонжеронов (в долях хорды) и такое относительное расстояние между нервюрами (в долях размаха),
при которых масса крыла минимальна. Используется гауссовский процесс и функция ожидаемого улучшения (EI) – это позволяет обойтись относительно небольшим
числом вызовов дорогого конечно-элементного расчёта.

/Что подаётся на вход.

for_ansys_{file_number}.npy – оттуда берутся начальные расположения силовой схемы.

Параметры оптимизации (те же, что и у thickness_optimizer).

/Что получается на выходе.

В папке "optimization_results_bayesian/" появляется файл spar_optimization_config_{idx}.txt с лучшим значением: spar1, spar2, relative spacing,
соответствующее количество нервюр и итоговая масса. Кроме того, в консоль выводится история итераций.

/Как запустить.

python spar_optimizer.py 1
Можно подкрутить параметры через аргументы (например, ограничить число нервюр).

- Как интерпретировать результаты

Масса – это масса полукрыла (одной консоли) после оптимизации. Для целого крыла удвойте.

Силовой фактор G – интеграл эквивалентного напряжения по объёму. Чем он меньше при той же нагрузке, тем рациональнее конструкция.

Коэффициент Ck – безразмерный показатель силового совершенства.

Толщины – записаны для каждого элемента отдельно.

- Замечания по практическому использованию

ANSYS должен быть установлен и прописан в системной переменной PATH, либо укажите полный путь в методе get_ansys_path() внутри thickness_optimizer.py.

MATLAB Engine для Python ставится командой pip install matlabengine – но перед этим нужно установить сам MATLAB с поддержкой Engine.

Если у вас нет MATLAB, можно заменить run_aero_batch.py на другой источник давлений – лишь бы файлы были в том же формате.

Все временные файлы (.db, .rst, .esav) автоматически удаляются, чтобы не засорять диск.
Если хотите сохранить промежуточные результаты – закомментируйте вызов cleanup_working_directory().

Количество потоков ANSYS по умолчанию равно числу ядер минус 1. Можно задать вручную через параметр "nproc" при вызове скриптов.
